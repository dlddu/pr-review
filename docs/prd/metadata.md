# PRD-1: PR 메타데이터 추출

> 리뷰에 영향을 미치는 3종 메타데이터(작업 유형 · 작성자 · 저장소)를 PR로부터 추출하고, 지식 베이스 조회 키로 정규화한다.

## 달성 가치
- **V3: 맥락 적응형 평가** — 추출된 메타데이터가 평가 보정의 1차 입력이 된다.
- **V4: 축적되는 메타데이터 지식** — 추출된 메타데이터가 지식 축적/조회의 키가 된다.

## Acceptance Criteria

### AC1-1: 작업 유형 분류
- **설명**: PR로부터 작업 유형을 식별하여 사용자가 정의한 작업 유형 집합 중 하나(또는 다중) 로 분류한다. 작업 유형이 불명확하면 "미분류"로 표시하고 PRD-6 공백 탐지의 대상이 되게 한다.
- **달성 가치**: V3, V4
- **검증 방법**: 다양한 PR 샘플(기능 추가 / 버그 수정 / 리팩토링 / 문서 등)에 대해 분류 결과가 정의된 유형 집합으로 매핑되는지 확인. 불명확한 PR은 "미분류"로 표시되는지 확인.

### AC1-2: PR 작성자 식별 (봇이면 실제 유저 추정)
- **설명**: PR 작성자를 식별한다. 작성자가 봇 계정이면 실제로 작업을 트리거한 유저를 추정하고, 추정 근거(커밋 author, 트리거 메타데이터, 연결된 이슈 담당자 등)를 함께 기록한다.
- **달성 가치**: V3, V4
- **검증 방법**: 사람이 만든 PR은 그대로 식별되는지, 봇이 만든 PR은 실제 유저가 추정되고 추정 근거가 제시되는지 확인. 추정이 불가능하면 "추정 불가"로 표시되고 공백으로 넘어가는지 확인.

### AC1-3: 저장소 식별
- **설명**: PR이 생성된 repository를 식별자로 추출한다.
- **달성 가치**: V3, V4
- **검증 방법**: PR로부터 저장소 식별자가 정확히 추출되어 지식 베이스의 저장소 키와 일치하는지 확인.

### AC1-4: 메타데이터의 지식 키 정규화
- **설명**: 추출된 3종 메타데이터를 PRD-4 지식 베이스의 조회 키 형식으로 정규화한다. (예: 작성자 별칭/봇 ID를 정규화된 유저 식별자로 통일)
- **달성 가치**: V4
- **검증 방법**: 동일 인물의 서로 다른 표기(봇 PR vs 직접 PR)가 동일한 키로 정규화되는지, 정규화 결과가 지식 베이스 키 스키마와 일치하는지 확인.
