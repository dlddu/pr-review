# PRD-4: 메타데이터 지식 베이스

> 메타데이터(작업 유형 · 작성자 · 저장소)를 키로 한 지식을 누적하고, 리뷰에 반영한다. 특정 메타데이터 슬라이스의 데이터가 부족하면 사용자에게 데이터 보강을 제안한다.

## 달성 가치
- **V4: 축적되는 메타데이터 지식** — 지식이 메타데이터별로 누적되고, 부족하면 보강을 제안한다.
- **V3: 맥락 적응형 평가** — 누적된 지식이 평가 보정의 재료가 된다.

## Acceptance Criteria

### AC4-1: 작업 유형별 지식 축적
- **설명**: 작업 유형마다 평가 기준의 variation과 허용 관용(tolerance) 범위를 누적 저장한다. (예: 리팩토링은 동작 변경이 없어야 한다는 기준 강화, 핫픽스는 일정 관용 확대 등)
- **달성 가치**: V4, V3
- **검증 방법**: 작업 유형 키로 조회 시 해당 유형의 기준 보정값과 관용 범위가 반환되는지, 새 리뷰 결과가 이 값으로 누적·갱신되는지 확인.

### AC4-2: 작성자별 지식 축적
- **설명**: 작성자마다 성향·성격·신뢰도 등을 누적한다. (예: 테스트를 꼼꼼히 붙이는 경향, 특정 영역 전문성, 과거 리뷰 통과율 기반 신뢰도)
- **달성 가치**: V4, V3
- **검증 방법**: 작성자 키(PRD-1에서 정규화됨)로 조회 시 성향/신뢰도 프로파일이 반환되고, 신뢰도가 리뷰 관용에 반영되는지 확인.

### AC4-3: 저장소별 지식 축적
- **설명**: 저장소마다 컨벤션과 코드 외적 규칙 등을 누적한다. (예: 네이밍/디렉터리 규칙, 리뷰어 지정 규칙, 배포 절차 등 코드 외적 맥락)
- **달성 가치**: V4, V3
- **검증 방법**: 저장소 키로 조회 시 컨벤션/규칙이 반환되고, 위반 시 해당 관점의 주장으로 연결되는지 확인.

### AC4-4: 데이터 부족 감지 및 보강 제안
- **설명**: 특정 메타데이터 슬라이스(작업 유형/작성자/저장소)의 누적 데이터가 임계치 미만이면, 리뷰 신뢰도가 낮아질 수 있음을 알리고 사용자에게 데이터 추가를 제안한다.
- **달성 가치**: V4
- **검증 방법**: 임계치 미만 슬라이스에 대해 "이 (작업 유형/작성자/저장소)에 대한 데이터가 부족합니다 — ~를 추가하시겠어요?" 형태의 제안이 생성되는지, 데이터가 충분한 슬라이스에서는 제안이 뜨지 않는지 확인.

### AC4-5: 지식의 리뷰 반영
- **설명**: 조회된 지식이 논리 엔진(PRD-5)의 주장 생성과 관용 판단의 입력으로 실제 사용된다.
- **달성 가치**: V4, V3
- **검증 방법**: 동일 PR을 지식 유/무 상태에서 각각 리뷰했을 때 관용 범위 또는 주장 구성이 달라지는지 확인(지식이 결론에 영향을 준다는 것을 입증).
