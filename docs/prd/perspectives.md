# PRD-3: 동적 평가 관점 관리

> 리뷰 평가의 기준이 되는 관점(perspective)을 동적으로 생성·삭제·적용한다. 관점은 논리 엔진(PRD-5)에서 주장을 만들어내는 출처가 되며, 양방향으로 추적 가능하다.

## 달성 가치
- **V3: 맥락 적응형 평가** — 어떤 관점을 적용하느냐에 따라 같은 PR도 다르게 평가된다.
- **V1: 신뢰할 수 있는 자동 리뷰** — 각 주장이 어떤 관점에서 나왔는지 추적할 수 있어야 결론을 검증할 수 있다.

## Acceptance Criteria

### AC3-1: 관점 생성
- **설명**: 새로운 평가 관점을 동적으로 생성한다. 각 관점은 식별자, 설명, 평가 기준(무엇을 보는가)을 가진다.
- **달성 가치**: V3, V1
- **검증 방법**: 생성된 관점이 식별자/설명/평가 기준을 갖추고 저장되며, 이후 리뷰에서 선택·적용 가능한 후보로 노출되는지 확인.

### AC3-2: 관점 삭제
- **설명**: 기존 관점을 삭제한다. 단, 과거 리뷰 기록이 참조하는 관점은 보존하여 과거 리뷰의 디버깅 가능성을 유지한다(삭제는 "이후 적용 중단"이며 과거 기록의 소급 삭제가 아니다).
- **달성 가치**: V3
- **검증 방법**: 삭제 후 해당 관점이 새 리뷰의 적용 후보에서 사라지는지, 그러나 과거 리뷰 기록에서 그 관점으로 만들어진 주장이 여전히 추적 가능한지 확인.

### AC3-3: 관점 적용
- **설명**: 특정 PR 리뷰에 관점들을 선택적으로 적용한다. 적용 여부는 메타데이터/컨텍스트에 따라 달라질 수 있다.
- **달성 가치**: V3, V1
- **검증 방법**: 적용된 관점만 논리 엔진의 주장 생성에 사용되고, 적용되지 않은 관점은 결론에 영향을 주지 않는지 확인.

### AC3-4: 관점–주장 추적성
- **설명**: 각 관점이 어떤 주장을 만들어냈는지, 각 주장이 어떤 관점에서 비롯됐는지 양방향으로 추적할 수 있다.
- **달성 가치**: V1
- **검증 방법**: 임의의 주장에서 관점으로, 임의의 관점에서 그 관점이 생성한 주장 목록으로 이동할 수 있는지 확인.
