# PRD-7: 리뷰 결과 표현 (인지 순서)

> 리뷰 결과를 사용자가 정보를 인지하는 고정된 순서로 표현한다. 사용자는 위에서부터 필요한 깊이까지만 내려가며, 한 번에 모든 정보를 떠안지 않는다.

## 달성 가치
- **V2: 최소 인지 부하로 리뷰 소비** — 정해진 인지 순서와 점진적 펼침으로 인지 부하를 최소화한다.

## 인지 순서 (정의)
1. **PR 내용 요약**
2. **리뷰의 결론**
3. **결론을 만들어내는 데 사용된 논리**
4. **논리에서 각 주장에 대한 관점과 근거**
5. **논리 사이의 비약/공백을 채우기 위해 필요한 행위·정보·관점 제안**
6. **공백을 채운 후 다시 리뷰 또는 리뷰 종결** (PR 머지 등의 행위는 하지 않음)

## Acceptance Criteria

### AC7-1: 고정 인지 순서 렌더링
- **설명**: 리뷰 결과를 위 1~6 순서로 표현한다. 섹션 순서가 임의로 뒤바뀌지 않는다.
- **달성 가치**: V2
- **검증 방법**: 출력 섹션의 순서가 정의된 1~6 순서와 정확히 일치하는지 확인.

### AC7-2: 점진적 펼침
- **설명**: 각 단계는 접힘/펼침 구조로, 사용자가 필요한 깊이까지만 펼쳐 볼 수 있다. 상위 단계만 보고도 핵심을 파악할 수 있고, 하위 단계는 on-demand로 펼친다.
- **달성 가치**: V2
- **검증 방법**: 결론(2단계)까지만 보고도 리뷰 핵심을 파악할 수 있는지, 논리/관점·근거(3~4단계)는 펼쳤을 때만 노출되는지 확인.

### AC7-3: PR 내용 요약 우선 표시
- **설명**: 가장 먼저 PR 내용 요약을 표시한다. 사용자가 리뷰를 읽기 전에 무엇에 대한 리뷰인지 먼저 인지하게 한다.
- **달성 가치**: V2
- **검증 방법**: 첫 섹션이 PR 내용 요약이며, 메타데이터/컨텍스트의 핵심이 요약에 반영되는지 확인.

### AC7-4: 단계 간 일관된 참조 (드릴다운)
- **설명**: 각 단계가 하위 단계로 일관되게 연결된다. 결론(2)의 각 주장에서 그 주장의 관점·근거(4)로, 공백 단계(5)에서 보완 제안으로 이동할 수 있다.
- **달성 가치**: V2, V1
- **검증 방법**: 결론의 임의 주장에서 해당 관점/근거로, 공백 항목에서 해당 보완 제안으로 막힘 없이 드릴다운되는지 확인.
