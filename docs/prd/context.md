# PRD-2: 리뷰 컨텍스트 수집

> 리뷰 시 확인해야 하는 컨텍스트(의도 · 관련 이슈 · 관련 스레드 · LOC · 일정/데드라인)를 수집한다. 일부 항목은 optional이며, 부재가 리뷰를 막지 않는다.

## 달성 가치
- **V3: 맥락 적응형 평가** — 수집된 컨텍스트가 평가 보정과 논리 엔진의 입력이 된다.

## Acceptance Criteria

### AC2-1: 의도 파악
- **설명**: PR의 의도를 수집한다(PR 제목·설명·커밋 메시지·연결된 이슈 등에서 추론). 의도는 리뷰의 기준점이므로 가능한 한 명시적으로 확정한다.
- **달성 가치**: V3
- **검증 방법**: 의도가 추출되면 리뷰 입력으로 전달되는지, 의도를 확정할 수 없으면 "의도 불명확"으로 표시되어 PRD-6 공백 탐지로 넘어가는지 확인.

### AC2-2: 관련 이슈 연결 (optional, 주로 Linear)
- **설명**: PR과 관련된 이슈(주로 Linear)를 연결한다. optional 항목이다.
- **달성 가치**: V3
- **검증 방법**: 이슈 참조가 있으면 연결되어 의도/일정 판단에 반영되는지, 없으면 "관련 이슈 없음"으로 명시되고 오류로 처리되지 않는지 확인.

### AC2-3: 관련 스레드 연결 (optional, 주로 Slack)
- **설명**: PR과 관련된 논의 스레드(주로 Slack)를 연결한다. optional 항목이다.
- **달성 가치**: V3
- **검증 방법**: 스레드 참조가 있으면 연결되어 의도/배경 판단에 반영되는지, 없으면 "관련 스레드 없음"으로 명시되는지 확인.

### AC2-4: 변경 규모(LOC) 측정
- **설명**: PR의 변경 규모를 LOC(추가/삭제 라인 수)로 수집한다.
- **달성 가치**: V3
- **검증 방법**: 추가/삭제 LOC가 수치로 기록되고, 규모가 관용 판단(PRD-4)과 리뷰 깊이 결정의 입력으로 전달되는지 확인.

### AC2-5: 일정·데드라인 컨텍스트
- **설명**: PR과 관련된 일정/데드라인 정보를 수집한다(연결된 이슈의 마감일, 릴리스 일정 등).
- **달성 가치**: V3
- **검증 방법**: 일정 정보가 있으면 기록되어 관용 판단(예: 데드라인 임박 시 허용 범위 조정)에 입력으로 전달되는지 확인.

### AC2-6: optional 컨텍스트의 부재 처리
- **설명**: optional 항목(관련 이슈·관련 스레드)의 부재가 리뷰 진행을 막지 않는다. 부재는 "없음"으로 명시되어 이후 공백 탐지가 필요 여부를 판단할 수 있게 한다.
- **달성 가치**: V3
- **검증 방법**: optional 항목이 전부 부재한 PR도 리뷰가 끝까지 진행되는지, 부재가 결론을 막는 오류가 아니라 명시적 "없음" 상태로 기록되는지 확인.
