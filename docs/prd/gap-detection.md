# PRD-6: 논리 공백 탐지 및 보완

> 리뷰 논리 체인에서 주장 간 비약 또는 근거 공백을 탐지하고, 공백을 채우기 위해 필요한 행위·정보·관점을 제안한다. 공백이 채워지면 재리뷰하거나 리뷰를 종결한다. **PR 머지 등 코드베이스를 바꾸는 행위는 수행하지 않는다.**

## 달성 가치
- **V5: 논리적 공백의 가시화와 보완** — 공백을 드러내고 보완 경로를 제안한다.
- **V1: 신뢰할 수 있는 자동 리뷰** — 비약 없는 논리만 결론으로 인정되도록 한다.

## Acceptance Criteria

### AC6-1: 비약/공백 탐지
- **설명**: 논리 체인을 검사하여 (a) 근거가 없는 주장, (b) 선행 주장 없이 도약한 결론, (c) optional이지만 결론에 필요한 컨텍스트의 부재를 공백으로 식별한다.
- **달성 가치**: V5
- **검증 방법**: 근거 없는 주장, 선행 주장이 비어 있는 결론, 결론에 필요한데 부재한 컨텍스트가 각각 공백으로 표시되는지 확인. 완결된 논리 체인에서는 공백이 0건으로 보고되는지 확인.

### AC6-2: 보완 제안 생성
- **설명**: 각 공백에 대해 그것을 채우기 위해 필요한 것을 **행위 / 정보 / 관점** 중 하나(또는 복수)로 분류해 구체적으로 제안한다. (예: "Linear 이슈 #123을 연결하세요"(행위), "이 함수의 호출처 영향 범위"(정보), "성능 관점을 추가 적용"(관점))
- **달성 가치**: V5
- **검증 방법**: 모든 공백마다 "필요한 것: (행위|정보|관점) — 구체 내용" 형태의 제안이 1건 이상 생성되는지, 제안이 막연한 표현이 아니라 실행 가능한 형태인지 확인.

### AC6-3: 보완 후 재리뷰
- **설명**: 사용자가 제안에 따라 행위/정보/관점을 보완하면, 논리 엔진(PRD-5)을 다시 돌려 갱신된 결론을 생성한다.
- **달성 가치**: V5
- **검증 방법**: 보완 입력 후 재리뷰가 트리거되어 주장/근거/결론이 갱신되고, 갱신 전후 차이가 추적 가능한지 확인.

### AC6-4: 리뷰 종결
- **설명**: 공백이 없거나, 사용자가 남은 공백을 감수하고 종결을 택하면 리뷰를 종결 상태로 기록한다. 종결은 "이 리뷰의 논리가 더 보완되지 않을 것"임을 의미할 뿐, 코드 변경을 수반하지 않는다.
- **달성 가치**: V5
- **검증 방법**: 공백 0건 상태 또는 사용자의 명시적 종결 선택 시 종결 상태가 기록되는지, 종결 사유(공백 없음 / 사용자 감수)가 남는지 확인.

### AC6-5: 코드베이스 변경 행위 금지 (경계 조건)
- **설명**: 본 제품은 PR 머지, 머지 승인, 브랜치 푸시 등 코드베이스를 바꾸는 어떤 행위도 수행하지 않는다. 리뷰 결과는 판단 보조일 뿐이다.
- **달성 가치**: V5, V1
- **검증 방법**: 시스템이 노출하는 동작 집합에 머지/승인/푸시/강제 종료형 코드 변경이 포함되지 않는지 확인. 종결/재리뷰는 리뷰 상태에만 영향을 주고 PR 상태(머지 여부)에는 영향을 주지 않는지 확인.
