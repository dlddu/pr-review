# 리뷰 렌즈(Lens) 제품 가치 문서

> 가칭입니다. 원하는 이름으로 바꿔도 무방하며, 이름이 바뀌어도 아래 가치 정의는 그대로 유효합니다.

## 제품 소유자
- 본인 (개인 프로젝트, 1인 소유)

## 제품 개요
PR 리뷰를 자동화하되, 자동화된 리뷰를 **신뢰**할 수 있도록 리뷰의 논리를 투명하게 드러내고 디버깅 가능하게 만드는 개인용 도구. 리뷰는 PR의 메타데이터와 컨텍스트에 맞춰 보정되며, 메타데이터를 키로 한 지식이 축적된다. 결론에 이르는 논리는 주장과 근거로 분리되어 기록되고, 논리의 비약·공백은 숨기지 않고 드러내어 보완 제안으로 이어진다. (PR 머지 등 코드베이스를 바꾸는 행위는 수행하지 않는다.)

---

## 제품 가치

### V1: 신뢰할 수 있는 자동 리뷰
- **유형**: 추상적
- **설명**: 리뷰가 자동으로 생성되더라도 사용자가 결론을 그대로 신뢰할 수 있어야 한다. 신뢰의 근거는 "결과가 그럴듯해서"가 아니라 "결론에 이르는 논리를 언제든 펼쳐서 검증할 수 있어서"이다. 모든 리뷰 결론은 주장(claim)과 근거(evidence)로 분해되어 추적 가능해야 한다.

### V2: 최소 인지 부하로 리뷰 소비
- **유형**: 구체적
- **설명**: 사용자가 리뷰를 이해할 때 정해진 인지 순서를 따른다 — (1) PR 내용 요약 → (2) 리뷰 결론 → (3) 결론을 만든 논리 → (4) 각 주장의 관점과 근거 → (5) 논리의 비약/공백을 채우기 위한 제안 → (6) 공백 보완 후 재리뷰 또는 종결. 사용자는 위에서부터 필요한 깊이까지만 내려가면 되므로 한 번에 모든 정보를 떠안지 않는다.

### V3: 맥락 적응형 평가
- **유형**: 추상적
- **설명**: 동일한 코드라도 작업 유형, 작성자, 저장소에 따라 평가가 달라진다. 리뷰는 PR의 메타데이터(작업 유형 · 작성자 · 저장소)와 수집된 컨텍스트(의도 · 관련 이슈 · 관련 스레드 · LOC · 일정/데드라인)에 맞춰 보정되며, 천편일률적인 일반 리뷰를 지양한다.

### V4: 축적되는 메타데이터 지식
- **유형**: 추상적
- **설명**: 리뷰가 반복될수록 메타데이터를 키로 하는 지식이 쌓인다. 작업 유형별 평가 기준 variation과 허용 관용 범위, 작성자의 성향·성격·신뢰도, 저장소의 컨벤션·코드 외적 규칙 등이 누적되어 다음 리뷰의 품질을 높인다. 특정 메타데이터 슬라이스의 데이터가 부족하면 시스템이 먼저 데이터 보강을 제안한다.

### V5: 논리적 공백의 가시화와 보완
- **유형**: 구체적
- **설명**: 리뷰의 논리에 비약이나 공백이 있으면 시스템이 이를 숨기지 않고 드러낸다. 공백을 채우기 위해 필요한 행위·정보·관점을 구체적으로 제안하고, 공백이 채워지면 재리뷰하거나 리뷰를 종결한다. 단, PR 머지 등 코드베이스를 바꾸는 행위는 수행하지 않는다.

---

## 가치 간 관계 (참고)
- V1(신뢰)은 V5(공백 가시화)와 V2(인지 순서)에 의해 뒷받침된다: 논리를 펼쳐 볼 수 있어야 신뢰가 성립한다.
- V3(맥락 적응)은 V4(지식 축적)를 입력으로 사용한다: 축적된 메타데이터 지식이 평가 보정의 재료가 된다.
- 모든 가치는 "PR을 머지하지 않는다"는 경계 안에서 동작한다. 본 제품은 판단을 돕는 도구이지, 코드베이스를 바꾸는 주체가 아니다.
