# J2: 논리 공백 보완 → 재리뷰

> 리뷰 결론의 논리에 드러난 비약·공백을, 시스템이 제안한 행위·정보·관점으로 채우고 재리뷰해 신뢰할 만한 리뷰로 만드는 여정. (PR 머지는 하지 않는다.)

## 페르소나
- **누가**: 리뷰 렌즈를 쓰는 개발자 본인
- **상황**: 리뷰 결론의 논리에 비약이나 근거 공백이 드러났고(주로 J1의 S3~S4에서 발견), 이대로는 결론을 신뢰하기 어렵다.
- **목표**: 시스템이 짚어 준 공백을 구체적 제안에 따라 채우고, 재리뷰로 논리를 완성해 결론의 신뢰도를 끌어올린다.

## 달성 가치
- **V5: 논리적 공백의 가시화와 보완** — 공백을 숨기지 않고 드러내고, 채우기 위해 필요한 것을 구체적으로 제안한 뒤, 채워지면 재리뷰하거나 종결한다.
- **V1: 신뢰할 수 있는 자동 리뷰** — 공백이 메워질수록 논리 체인이 완전해져 결론을 신뢰할 근거가 강해진다.

## 여정 단계

### S1: 비약·공백 인지
- **사용자**: 리뷰가 드러낸 논리 공백 지점(근거 없는 주장 / 도약한 결론 / 결론에 필요한데 빠진 컨텍스트)을 확인한다.
- **시스템**: 논리 체인을 검사해 공백을 명시적으로 표시한다.
- **관련 PRD/AC**: PRD-6 AC6-1
- **시각화 (mockup)**: 🔲 TBD

### S2: 보완 제안 확인
- **사용자**: 각 공백을 채우려면 무엇이 필요한지 — **행위 / 정보 / 관점** 중 무엇인지 — 제안을 검토한다.
- **시스템**: 공백별로 필요한 것을 행위·정보·관점으로 분류해 구체적으로 제안한다. (예: "이슈 #123 연결"(행위), "이 함수의 호출처 영향 범위"(정보), "성능 관점 추가"(관점))
- **관련 PRD/AC**: PRD-6 AC6-2
- **시각화 (mockup)**: 🔲 TBD

### S3: 정보·관점 보강
- **사용자**: 제안에 따라 누락 컨텍스트를 연결하거나 새 평가 관점을 추가·적용한다.
- **시스템**: 추가된 컨텍스트/관점을 리뷰 입력으로 받아들인다.
- **관련 PRD/AC**: PRD-2(컨텍스트 수집), PRD-3 AC3-1·AC3-3(관점 생성·적용)
- **시각화 (mockup)**: 🔲 TBD

### S4: 재리뷰
- **사용자**: 보강을 반영해 다시 리뷰를 돌리고, 공백이 해소됐는지 확인한다.
- **시스템**: 논리 엔진을 재실행해 갱신된 결론을 생성한다.
- **관련 PRD/AC**: PRD-6 AC6-3, PRD-5 AC5-1·AC5-4
- **분기**: 새 공백이 또 보이면 → S1로 반복 / 충분하면 → S5
- **시각화 (mockup)**: 🔲 TBD

### S5: 리뷰 종결
- **사용자**: 공백이 충분히 메워졌거나 남은 공백을 감수하기로 하면 리뷰를 종결한다.
- **시스템**: 종결 상태를 기록한다. 종결은 "논리가 더 보완되지 않음"을 뜻할 뿐 **코드 변경·머지를 수반하지 않는다.**
- **관련 PRD/AC**: PRD-6 AC6-4·AC6-5
- **시각화 (mockup)**: 🔲 TBD
