## 가치 커버리지 메모 (설계 의도)
- **V1(신뢰)** 은 한 PRD가 아니라 관점 추적성(PRD-3) + 주장·근거 분리(PRD-5) + 공백/머지금지(PRD-6) + 드릴다운(PRD-7)의 조합으로 달성된다. "논리를 펼쳐 볼 수 있어야 신뢰가 성립한다"는 가치 정의를 반영.
- **V3(맥락 적응)** 은 입력(PRD-1 메타데이터, PRD-2 컨텍스트) + 보정 재료(PRD-4 지식) + 적용(PRD-3 관점)으로 달성된다.
- **"PR 머지를 하지 않는다"** 는 경계 조건은 AC6-5에 명시적 검증 항목으로 박아 두었다. 이후 어떤 기능을 추가하더라도 이 경계가 깨지지 않는지 AC6-5로 회귀 검증할 것.

## 다음에 가치/PRD가 바뀌면 할 일 (유지보수 규칙)
1. 가치가 추가/수정/삭제되면 → 영향받는 PRD·AC·테스트의 정합성을 재검증하고 이 매트릭스를 갱신.
2. PRD에 AC가 추가되면 → 해당 AC를 가치에 연결하고, 대응 테스트 시나리오를 추가(미검증 AC 발생 방지).
3. 관점/지식 스키마가 바뀌면 → PRD-1(키 정규화)·PRD-4(지식 베이스)·PRD-5(주장 귀속)의 연결을 재확인.
4. 매 산출물 생성/수정 시 → 이 추적 문서의 "현재 상태 요약"과 "변경 이력"을 함께 갱신.
