# 테스트 문서: 동적 평가 관점 관리

## 검증 대상 AC
- AC3-1: 관점 생성 (PRD-3)
- AC3-2: 관점 삭제 — 과거 기록 보존 (PRD-3)
- AC3-3: 관점 적용 (PRD-3)
- AC3-4: 관점–주장 추적성 (PRD-3)

## 테스트 시나리오

### 시나리오 1: 새 관점 생성
- **사전 조건**: 관점 목록이 비어 있거나 일부 존재.
- **실행 단계**: "성능" 관점을 식별자/설명/평가 기준과 함께 생성.
- **기대 결과**: 생성된 관점이 저장되고 이후 리뷰의 적용 후보로 노출된다.
- **검증 AC**: AC3-1

### 시나리오 2: 관점 삭제 후 과거 기록 보존
- **사전 조건**: "성능" 관점으로 만든 주장이 포함된 과거 리뷰가 존재.
- **실행 단계**: "성능" 관점 삭제 → 새 리뷰의 적용 후보 확인 + 과거 리뷰 추적 확인.
- **기대 결과**: 새 리뷰 후보에서는 사라지지만, 과거 리뷰에서 그 관점으로 만들어진 주장은 여전히 추적 가능하다.
- **검증 AC**: AC3-2

### 시나리오 3: 선택적 관점 적용
- **사전 조건**: 관점 A·B·C가 존재. 특정 PR에 A·C만 적용.
- **실행 단계**: 리뷰 실행 → 주장 생성에 사용된 관점 확인.
- **기대 결과**: A·C에서만 주장이 생성되고 B는 결론에 영향을 주지 않는다.
- **검증 AC**: AC3-3

### 시나리오 4: 관점↔주장 양방향 추적
- **사전 조건**: 관점이 적용되어 주장이 생성된 리뷰 존재.
- **실행 단계**: 임의 주장 → 관점 이동, 임의 관점 → 생성 주장 목록 이동 시도.
- **기대 결과**: 양방향 모두 막힘 없이 추적된다.
- **검증 AC**: AC3-4
