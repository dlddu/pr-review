# 테스트 문서: 리뷰 결과 표현 (인지 순서)

## 검증 대상 AC
- AC7-1: 고정 인지 순서 렌더링 (PRD-7)
- AC7-2: 점진적 펼침 (PRD-7)
- AC7-3: PR 내용 요약 우선 표시 (PRD-7)
- AC7-4: 단계 간 일관된 참조(드릴다운) (PRD-7)

## 테스트 시나리오

### 시나리오 1: 1~6 순서 렌더링
- **사전 조건**: 결론·논리·관점·근거·공백 제안이 모두 존재하는 리뷰.
- **실행 단계**: 결과 표현 → 섹션 순서 확인.
- **기대 결과**: (1) PR 요약 → (2) 결론 → (3) 논리 → (4) 관점·근거 → (5) 공백 제안 → (6) 재리뷰/종결 순서로 표시된다.
- **검증 AC**: AC7-1

### 시나리오 2: 결론까지만 보고 핵심 파악
- **사전 조건**: 다단계 리뷰 결과.
- **실행 단계**: 상위 단계(요약·결론)만 펼쳐서 확인.
- **기대 결과**: 결론까지만 봐도 리뷰 핵심을 파악할 수 있고, 논리/관점·근거는 펼쳤을 때만 노출된다.
- **검증 AC**: AC7-2

### 시나리오 3: PR 요약 우선 표시
- **사전 조건**: 메타데이터·컨텍스트가 수집된 리뷰.
- **실행 단계**: 결과 표현 → 첫 섹션 확인.
- **기대 결과**: 첫 섹션이 PR 내용 요약이며 메타데이터/컨텍스트 핵심이 반영된다.
- **검증 AC**: AC7-3

### 시나리오 4: 결론→근거, 공백→제안 드릴다운
- **사전 조건**: 주장과 공백이 포함된 리뷰 결과.
- **실행 단계**: 결론의 임의 주장에서 관점/근거로, 공백 항목에서 보완 제안으로 이동.
- **기대 결과**: 두 드릴다운 경로 모두 막힘 없이 연결된다.
- **검증 AC**: AC7-4
