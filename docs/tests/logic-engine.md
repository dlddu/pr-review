# 테스트 문서: 리뷰 논리 엔진

## 검증 대상 AC
- AC5-1: 결론 도출 (PRD-5)
- AC5-2: 주장–근거 분리 기록 (PRD-5)
- AC5-3: 주장의 관점 귀속 (PRD-5)
- AC5-4: 논리 체인 구성 (PRD-5)
- AC5-5: 단계별 디버깅 뷰 (PRD-5)

## 테스트 시나리오

### 시나리오 1: 결론은 반드시 주장에 의해 뒷받침됨
- **사전 조건**: 관점·컨텍스트·지식이 입력된 리뷰.
- **실행 단계**: 결론 생성 → 결론과 주장의 연결 확인.
- **기대 결과**: 모든 결론이 최소 1개 주장과 연결된다. 주장 없는 "맨몸 결론"이 생성되지 않는다.
- **검증 AC**: AC5-1

### 시나리오 2: 주장과 근거의 분리 저장
- **사전 조건**: 주장이 생성된 리뷰.
- **실행 단계**: 임의 주장의 저장 구조 확인.
- **기대 결과**: 각 주장이 고유 식별자와 근거 집합을 갖고, 근거가 주장과 분리되어 독립 조회·수정 가능하다.
- **검증 AC**: AC5-2

### 시나리오 3: 주장의 관점 귀속 확인
- **사전 조건**: 관점이 적용되어 주장이 생성된 리뷰.
- **실행 단계**: 각 주장의 관점 귀속 정보 확인.
- **기대 결과**: 모든 주장이 하나 이상의 관점에 귀속되고 관점→주장 역추적이 가능하다.
- **검증 AC**: AC5-3

### 시나리오 4: 결론에서 선행 주장까지 역추적
- **사전 조건**: 다단계 논리 체인이 구성된 리뷰.
- **실행 단계**: 결론에서 역방향으로 체인을 따라가기.
- **기대 결과**: 결론에서 모든 선행 주장까지 도달 가능하고 의존 관계가 명시된다.
- **검증 AC**: AC5-4

### 시나리오 5: 단계별 분리 조회로 인지 부하 최소화
- **사전 조건**: 여러 단계의 주장/근거가 있는 리뷰.
- **실행 단계**: 한 단계만 펼쳐서 조회.
- **기대 결과**: 한 단위에서 하나의 단계(주장+그 근거)만 펼쳐지고, 사용자가 원하는 단계까지만 내려갈 수 있다.
- **검증 AC**: AC5-5
