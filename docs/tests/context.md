# 테스트 문서: 리뷰 컨텍스트 수집

## 검증 대상 AC
- AC2-1: 의도 파악 (PRD-2)
- AC2-2: 관련 이슈 연결 — optional (PRD-2)
- AC2-3: 관련 스레드 연결 — optional (PRD-2)
- AC2-4: 변경 규모(LOC) 측정 (PRD-2)
- AC2-5: 일정·데드라인 컨텍스트 (PRD-2)
- AC2-6: optional 컨텍스트의 부재 처리 (PRD-2)

## 테스트 시나리오

### 시나리오 1: 의도가 명확한 PR
- **사전 조건**: 제목/설명/연결 이슈에 의도가 드러나는 PR.
- **실행 단계**: 컨텍스트 수집 → 의도 항목 확인.
- **기대 결과**: 의도가 추출되어 리뷰 입력으로 전달된다.
- **검증 AC**: AC2-1

### 시나리오 2: 의도를 확정할 수 없는 PR
- **사전 조건**: 설명이 비어 있고 변경만으로 의도 추론이 어려운 PR.
- **실행 단계**: 의도 추출 시도 → 상태 확인.
- **기대 결과**: "의도 불명확"으로 표시되어 PRD-6 공백 탐지로 넘어간다.
- **검증 AC**: AC2-1

### 시나리오 3: Linear 이슈가 연결된 PR
- **사전 조건**: PR 설명/브랜치명에 Linear 이슈 참조 존재.
- **실행 단계**: 관련 이슈 연결 실행.
- **기대 결과**: 이슈가 연결되어 의도/일정 판단에 반영된다.
- **검증 AC**: AC2-2

### 시나리오 4: Slack 스레드가 연결된 PR
- **사전 조건**: 관련 Slack 스레드 링크가 존재하는 PR.
- **실행 단계**: 관련 스레드 연결 실행.
- **기대 결과**: 스레드가 연결되어 배경/의도 판단에 반영된다.
- **검증 AC**: AC2-3

### 시나리오 5: LOC 측정
- **사전 조건**: 추가/삭제 라인이 섞인 PR.
- **실행 단계**: LOC 측정 실행.
- **기대 결과**: 추가/삭제 LOC가 수치로 기록되고 관용 판단·리뷰 깊이 입력으로 전달된다.
- **검증 AC**: AC2-4

### 시나리오 6: 데드라인 임박 PR의 일정 반영
- **사전 조건**: 연결 이슈에 임박한 마감일이 있는 PR.
- **실행 단계**: 일정 수집 → 관용 판단 입력 확인.
- **기대 결과**: 일정 정보가 기록되어 관용 판단(허용 범위 조정)의 입력으로 전달된다.
- **검증 AC**: AC2-5

### 시나리오 7: optional 컨텍스트가 모두 부재한 PR
- **사전 조건**: 관련 이슈·관련 스레드가 모두 없는 PR.
- **실행 단계**: 컨텍스트 수집 → 리뷰 진행 여부 확인.
- **기대 결과**: 부재 항목은 "없음"으로 명시되고, 리뷰가 끝까지 진행된다(부재가 오류로 처리되지 않는다).
- **검증 AC**: AC2-6
