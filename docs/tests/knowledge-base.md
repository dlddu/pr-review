# 테스트 문서: 메타데이터 지식 베이스

## 검증 대상 AC
- AC4-1: 작업 유형별 지식 축적 (PRD-4)
- AC4-2: 작성자별 지식 축적 (PRD-4)
- AC4-3: 저장소별 지식 축적 (PRD-4)
- AC4-4: 데이터 부족 감지 및 보강 제안 (PRD-4)
- AC4-5: 지식의 리뷰 반영 (PRD-4)

## 테스트 시나리오

### 시나리오 1: 작업 유형별 관용 범위 누적/조회
- **사전 조건**: "리팩토링" 유형에 대한 리뷰가 여러 건 누적됨.
- **실행 단계**: "리팩토링" 키로 지식 조회.
- **기대 결과**: 해당 유형의 기준 보정값(예: 동작 변경 금지 강화)과 관용 범위가 반환되고, 새 리뷰 결과로 갱신된다.
- **검증 AC**: AC4-1

### 시나리오 2: 작성자 신뢰도의 관용 반영
- **사전 조건**: 특정 작성자에 대한 성향/신뢰도 프로파일이 누적됨.
- **실행 단계**: 작성자 키로 조회 → 신뢰도가 관용에 반영되는지 확인.
- **기대 결과**: 신뢰도 높은 작성자에게는 관용이 확대되는 등 프로파일이 리뷰에 반영된다.
- **검증 AC**: AC4-2

### 시나리오 3: 저장소 컨벤션 위반 연결
- **사전 조건**: 특정 저장소에 네이밍/디렉터리 컨벤션이 누적됨. 컨벤션을 위반한 PR 입력.
- **실행 단계**: 저장소 키로 조회 → 위반 탐지 → 주장 연결 확인.
- **기대 결과**: 컨벤션이 반환되고 위반이 해당 관점의 주장으로 연결된다.
- **검증 AC**: AC4-3

### 시나리오 4: 데이터 부족 슬라이스의 보강 제안
- **사전 조건**: 신규 작성자(누적 데이터 임계치 미만).
- **실행 단계**: 해당 작성자 PR 리뷰 → 보강 제안 발생 여부 확인.
- **기대 결과**: "이 작성자에 대한 데이터가 부족합니다 — ~를 추가하시겠어요?" 제안이 생성된다. 데이터가 충분한 슬라이스에서는 제안이 발생하지 않는다.
- **검증 AC**: AC4-4

### 시나리오 5: 지식 유/무에 따른 결론 차이
- **사전 조건**: 동일 PR. 케이스 A는 관련 지식 없음, 케이스 B는 지식 충분.
- **실행 단계**: 두 케이스를 각각 리뷰 → 관용 범위·주장 구성 비교.
- **기대 결과**: 지식이 있는 케이스에서 관용/주장이 달라진다(지식이 결론에 실제로 영향을 준다).
- **검증 AC**: AC4-5
